Best use of AI/ML API
The scoring call is load-bearing

No finding gets a verdict
until the AI/ML API returns.

@RiskScorer sends every candidate finding to the AI/ML API, which weighs three dimensions and returns a structured exposure score.

Severity
Statutory penalty & litigation exposure
Likelihood
How clearly the evidence establishes a violation
Jurisdiction
Whether the rule actually attaches here
scoreexposure() → AI/ML API
# PAYTRANS-CA-SB1162

  "exposure_score": 88,
  "severity": "severe",
  "likelihood": "high",
  "jurisdiction_attaches": true,

Adversarial review
Agents check each other

@Counsel can bounce a weak
finding back to @PolicyAgent.

Before signing, @Counsel adversarially re-examines every Critical finding. If the evidence is thin or the jurisdiction doesn't clearly attach, it sends the finding back into the room for re-examination — a visible, defensible review loop rather than a single-pass rubber stamp.

band room · the re-loop

  evidence is thin, no blanket exclusion in the posting.
@PolicyAgent agreed — downgrading to Suggestion. Re-scored.
@Counsel finalized. Writing audit.md.
